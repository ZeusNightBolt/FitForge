**Asset strategy verdict (per the hard constraints):** confirmed — **100% self-authored inline SVG React components + build-time-rendered PNGs**. No external hosts, no photo/GIF licensing, no runtime fetches; everything ships in the static export and works offline. Research (§1) backs this: the two most-loved "visual intelligence" patterns in this space — MuscleWiki's clickable body map and Fitbod's front/back recovery heatmap — are *vector anatomy silhouettes*, not photography. We can match the category leaders' signature visuals with zero binary assets in git (only the build-rendered favicon/OG PNGs are binary, and they're generated by a script).

### 1.2 Synthesis — what makes a great, easy-to-use PT app

1. **One dark canvas, one accent.** Nike/Freeletics/Caliber/Fitbod/Hevy all converge on near-black + white type + a single saturated accent reserved for the primary action. Restraint reads premium; rainbow reads cheap. → Gold gets the Freeletics-blue role: scarce, always meaningful.
2. **The body silhouette does triple duty** — navigation (MuscleWiki), exercise metadata (wger/Strong/Fitbod highlighted-muscle diagrams), analytics (Hevy/Jefit/Fitbod heat/recovery maps). One well-built front/back SVG with per-muscle ids pays off three times. This is the single highest-leverage asset we can build, and it's exactly what a static offline export can ship where video apps can't.
3. **The winning exercise-visual stack is layered:** anatomy diagram + demo + numbered steps. We ship layers 1 and 3 at launch (diagram + steps + cues); animated demos are a later optional layer.
4. **Local-first is a differentiator, not a compromise.** The most-loved free tools (FitNotes, Liftosaur, MuscleWiki web, Strong's skip-account, wger guest) let you do the core thing before any identity exists. Liftosaur proves the exact architecture (browser storage + optional sync). → "Local Mode" is a brag, front and center.
5. **Value before any wall:** ≤6 focused questions, then immediately *show* the personalized artifact (Fitbod's recovery map / generated workout). → Our `plan_preview` step is the money moment; make it gorgeous.
6. **Equipment UX is universally underdeveloped** — mostly flat chips; only Fitbod illustrates, only Liftosaur models plates. An illustrated picker with Home/Commercial/Bodyweight presets beats almost everyone. → P0.
7. **Mid-workout minimalism:** current set, last-time numbers, rest timer, one big done button, nothing else, huge targets.
8. **Show intelligence visually** (Fitbod heatmap, Caliber Strength Score): one glanceable picture > a wall of charts — and all computable offline from logged sets.
9. **Exercise detail IA:** Strong's About/History/Charts/Records is the cleanest reference pattern.
10. **Trust is a design feature.** Ad-free, no paywalled basics (Jefit's dark mode, MFP's scanner are the anti-patterns), export your data anytime. Local Mode + JSON export = our trust story.

## 2. Brand system — "Forged Gold" on ink navy

Dark is the **default brand experience** (`:root` = dark). A light "Ivory" variant is specified for `prefers-color-scheme: light` users, using bronze (not raw gold) for interactive text so AA holds on light surfaces. Token *names* are unchanged from the current `globals.css` so every existing `bg-accent` / `text-muted-foreground` / `border-border` call site restyles for free; new tokens are additive.

### 2.4 Depth, radius, motion

- **Elevation = borders + soft shadow, not lightness alone.** Cards: `bg-surface-2` + `border border-border` + `shadow-card` + the inset top hairline (baked into `--elev-card`) that reads as machined edge-light.
- **Gold glow** (`shadow-glow`) is reserved for: the primary CTA on landing/onboarding, the active workout's current set card, and PR moments. Never two glows on screen.
- **Gradient hairline** (`.border-gradient-gold`) for "premium" containers: plan-preview card, PR cards, the Today hero card.
- Radii: cards 20, fields/buttons 14, tiles 10, chips pill.
- Motion: 150–200ms ease-out on hover/press; 250ms cubic-bezier(0.32,0.72,0,1) sheet slide-ups; a single signature moment — the **spark**: on set-complete and onboarding-finish, a 500ms 4-point gold star scales in/out at the interaction point (CSS only, honors `prefers-reduced-motion`, already globally disabled by the existing reduced-motion rule).

## 3. Logo, favicon, OG share card

### 3.1 Logo concept — "The Anvil Bar"

A barbell resting across an anvil: training (bar) + forging (anvil) in one silhouette, topped by a 4-point spark. Reads at 16px, authors in ~10 SVG shapes.

**Mark geometry (viewBox `0 0 64 64`, gold `var(--gradient-gold)` fill on transparent/dark):**

1. **Bar:** `<rect x="4" y="17" width="56" height="6" rx="3"/>` — full-width horizontal bar.
2. **Plates:** `<rect x="10" y="10" width="6" height="20" rx="2"/>` and mirrored `<rect x="48" .../>` — one plate pair straddling the bar (drawn over it, same gold; a 1.5px `#0A0D14` gap-stroke separates plate from bar so the silhouette stays legible when flat).
3. **Anvil top face:** `<rect x="17" y="28" width="30" height="7" rx="2"/>` with a left **horn**: `<path d="M17 28 C11 28 8 30 6 34 L17 35 Z"/>` (pointed forge horn poking left).
4. **Anvil waist:** trapezoid `<path d="M25 35 L39 35 L36 44 L28 44 Z"/>`.
5. **Anvil base:** `<rect x="23" y="44" width="18" height="6" rx="2"/>`.
6. **Spark:** 4-point star centered `(50, 8)`, `<path d="M50 3 L51.8 6.2 L55 8 L51.8 9.8 L50 13 L48.2 9.8 L45 8 L48.2 6.2 Z"/>` in flat `#F6D883` (the brightest gold stop) — the "strike" happening where bar meets effort.

All shapes single gold gradient (`<linearGradient id="ffgold" x1="0" y1="0" x2="1" y2="1">` stops `#F6D883`/`#E4B84D` 45%/`#B8862C`); the spark flat-light-gold on top. Monochrome variant: everything `currentColor`.

### 4.1 The muscle-anatomy map (the exercise "image")

**Design language:** stylized "engineering-drawing anatomy", not medical. Two standing silhouettes (front & back), viewBox `0 0 200 440` each, drawn as: body outline stroke `var(--body-outline)` 1.5px, no fill; each muscle a closed simplified polygon-ish path filled `var(--muscle-base)` with `var(--muscle-line)` 1px separation strokes and ~1.5px visual gaps between muscles (the "plating" look — reads forged/armored, on-brand). Highlighted muscles fill gold. This is exactly the MuscleWiki/Fitbod pattern that users already understand, and it doubles as the recovery/volume heatmap surface later.

**Data model (`paths.ts`):**

```ts
export type MuscleView = 'front' | 'back';
export interface MusclePath {
  view: MuscleView;
  /** path data for the RIGHT-side (or center) shape only */
  d: string;
  side: 'right' | 'center';   // 'right' shapes are auto-mirrored to the left
}
export const MUSCLE_PATHS: Record<MuscleSlug, MusclePath[]> = { /* … */ };
```

Author only right-half + center shapes; `MuscleMap` mirrors `side:'right'` paths with `<use transform="scale(-1,1) translate(-200,0)">` — guarantees symmetry and halves authoring effort. Multi-region muscles (e.g. quads' three visible heads) are multiple `MusclePath` entries under one slug.

**View placement for all 20 seed slugs** (from `seed/data/muscles.json` `is_front`, refined for real silhouettes):

| View | Muscles (slug) |
|---|---|
| **Front** | `pecs` (2 fan-shaped plates), `front-delts` (shoulder caps), `side-delts` (outer cap sliver — front), `biceps`, `forearms` (front + back entries), `abs` (single center path with 3 crossline strokes suggesting the wall), `obliques` (side panels flanking abs), `hip-flexors` (small crease wedges), `quads` (3 lobes: rectus + vastus med/lat), `adductors` (inner-thigh teardrops), plus a thin `traps` collar sliver |
| **Back** | `traps` (kite from neck to mid-back), `rear-delts`, `rhomboids` (between shoulder blades, under traps), `lats` (two wings), `lower-back` (twin erector columns), `triceps`, `forearms` (again), `glute-max` (2 plates), `glute-med` (upper-outer hip wedges above glute-max), `hamstrings` (2 long straps), `calves` (2 diamonds) |

(`side-delts`, `forearms`, `traps` appear in both views; the component highlights every path registered for a slug, whatever view is shown.)

**Component API — frozen contract (WS-C ships it, WS-F consumes it; do not change signatures without updating both):**

```tsx
interface MuscleMapProps {
  primary?: MuscleSlug[];               // fill: var(--accent), fill-opacity .95
  secondary?: MuscleSlug[];             // fill: var(--accent), fill-opacity .38
  view?: MuscleView | 'both' | 'auto';  // 'auto' (default): view containing the most
                                        // primary paths; ties → front. 'both': side by side
  heat?: Partial<Record<MuscleSlug, number>>; // 0..1 → gold fill-opacity .15+.75x
                                        // (weekly-volume / recovery heatmap mode;
                                        //  overrides primary/secondary when present)
  height?: number;                      // px, default 260; width follows 200:440 ratio
  interactive?: boolean;                // muscles become <a role="button" tabIndex=0>,
  onMuscleClick?: (slug: MuscleSlug) => void;  // hover/focus: stroke var(--accent)
  labels?: boolean;                     // tiny leader-line labels for highlighted muscles
  className?: string;
}
```

- **A11y:** `role="img"` + `aria-label` auto-composed: `"Muscles worked — primary: Quads; secondary: Glutes, Lower Back"`. Interactive mode: each muscle path wrapped in a focusable element with `aria-label` = muscle name. Detail pages must also render the text chip list (already exists — "Muscles worked" copy is e2e-load-bearing).
- **`MuscleMapThumb`**: single auto-view, `height≈56`, no strokes on unhighlighted muscles (outline + gold fills only) — the "image" in every exercise list card. Cheap: same path data, one component.
- **Authoring workflow:** the Fable visual agent authors `outline.ts` + `paths.ts` as hand-tuned path data (target ≤ 6KB gzipped total); verify by rendering all-20-highlighted in both views in a scratch page and eyeballing against the table above.

**Usage map:** exercise cards (thumb), exercise detail (`view="both"`, labels on), onboarding `exercise_prefs` + exclusions ("protect body areas" becomes tappable `interactive` map), exercises page **muscle filter** (tap a muscle → filter list), progress **weekly volume heatmap** (`heat` mode), OG card, landing hero.
